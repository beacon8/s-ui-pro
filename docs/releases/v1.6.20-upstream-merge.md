## 2. 合并基线

| 范围 | 起点 | 目标 |
| --- | --- | --- |
| 上游后端 | `v1.4.1` / `1f393fc37fae50cafe6029bdc7620617e1576e50` | `v1.5.3` / `843c640ba4842d5bde4e179c7da50a3d5052b67c` |
| 上游前端 | `f65f58efbd12b8c1eed6f3ca15a09f05746b6819` | `d2b56b78944bfc425c59d92e02e88188d3f78656` |
| Pro 导入上游快照 | `3eb70ee9ed659fef8c042d09ce4d80b9630c316d` | — |
| Pro 合并前版本 | `db315f03246111616b1597cf08b3b44faa58b4a9` (`v1.6.19`) | `e26e35a` (`v1.6.20`) |

未直接执行普通 `git merge`，原因是：

- 上游后端仓库把 `frontend` 作为 Git submodule。
- Pro 仓库直接保存完整前端源码。
- 两边仓库结构不同，直接合并会把前端目录误判为 submodule 变更。

本次采用的方式：

1. 从共同基线生成上游后端累计三方补丁。
2. 从对应前端基线生成上游前端累计三方补丁。
3. 分别应用到 Pro，保留 Pro 自有功能并手工解决冲突。
4. 排除不应覆盖 Pro 的上游仓库元数据、工作流、文档和版本文件。
5. 只同步已发布的上游稳定版 `v1.5.3`，没有混入上游 `main` 的未发布提交。

## 4. 保留的 s-ui-pro 功能

本次同步不是用上游代码覆盖 Pro。下列 Pro 功能继续保留：

### 节点、规则与客户端管理

- 本地粘贴节点并解析导入。
- 单请求 `newbulk` 批量创建节点。
- 路由规则批量导入，冲突项跳过并立即写入数据库。
- 客户端上传、下载限速，支持 `mbps`、`kbps`、`bps`。
- 共享 token bucket 限速和运行中热更新。
- 客户端批量新增限速配置。
- Top Users 支持时间周期和上传/下载方向筛选。

### 聚合订阅

- 多客户端 JSON 聚合订阅。
- 支持客户端筛选、API key、随机路径和无效路径 404 防护。
- 聚合订阅上限由 2,000 提高到 10,000。

### 列表与前端交互

- Outbounds、Rules 等页面保留分页、搜索和表格化管理。
- 保留复制分享链接功能。
- 前端复制对话框继续用客户端用户名重写节点名称，包括 VMess `ps`。
- 注意：合并后后端直接生成的原始链接遵循上游 Remark 逻辑；前端复制对话框仍执行用户名重写。

### Pro 界面

- 保留 Ant Design 主题。
- 保留响应式 Dashboard。
- 保留可持久化的 Push 侧边栏。
- 保留表格溢出、标签显示和移动端布局修复。
- Pro 早期加入的整套 SSL 自动化已在 `v1.6.7` 回滚，本版本不包含该功能。
- 本版本包含的是上游提供的面板/订阅 TLS 证书热重载和证书 pin 能力。

### 稳定性与批量编辑

- 后续数据安全分支已把逐表批量复制替换为 SQLite `VACUUM INTO` 一致快照；详细失败边界见 [`docs/database-safety.md`](../database-safety.md)。已发布的 `v1.6.20` 资产是否包含该改动仍应以 tag/commit 为准。
- 保留 Outbound 批量编辑：多选、多字段搜索、多行 `ip:port:username:password`。
- 密码中允许包含冒号。
- 支持导入预览、密码遮罩、行数校验和配置热替换。
- 保留选择状态、弹窗状态及数据库锁相关修复。

## 5. 合并时新增的兼容修复

### 数据库迁移兼容

Pro 的版本号已经高于上游 `1.5.x`。如果继续使用单纯的版本大小判断，TLS 和 bcrypt 等上游迁移会被错误跳过。

`cmd/migration/main.go` 已调整为：版本不一致时执行可重复、幂等的迁移，而不是只在当前版本号小于目标版本时执行。

### 管理员密码兼容

- 使用 `bcrypt.Cost` 严格判断是否为有效 bcrypt 哈希。
- 避免把仅以 `$2` 开头的普通明文密码误判成哈希，从而造成管理员无法登录。
- 新密码只保存 bcrypt 哈希，不能反向显示或恢复。

### 客户端批量编辑兼容

批量保存客户端时，前端没有提交的服务器管理字段会继续保留，包括：

- `Config`、`Links`
- 创建时间、在线时间、延迟
- 自动重置配置
- 上传、下载及总流量计数

这样可避免使用 Pro 批量编辑后清空上游新增字段。

### 外部内容解析

- 粘贴 JSON 或订阅内容时先清除首尾空白字符。
- 避免换行、空格导致有效 JSON/订阅解析失败。

### 新增回归测试

- `cmd/migration/main_test.go`
- `cmd/migration/tls_test.go`
- `service/client_test.go`
- `util/password_test.go`
- `util/subToJson_test.go`

## 6. 管理员密码行为变化

数据库中的 `Password: <hashed, not recoverable>` 是预期行为，不是显示故障。

- 从本版本开始，管理员密码保存为 bcrypt 哈希。
- 哈希不能解密，所以系统不能显示原密码。
- 忘记密码时只能重置，例如：

```bash
/usr/local/s-ui/sui admin -username admin -password '新密码'
```

手动替换二进制后应执行：

```bash
/usr/local/s-ui/sui migrate
```

使用项目 `install.sh` 安装或升级时，脚本会自动执行迁移。

## 10. 已知遗留风险

以下问题不是本次发布阻断项，但后续修改相关代码时应优先检查：

- `pendingHotSwaps` 是未加锁的包级全局变量；并发管理员保存操作可能发生竞争。
- “重置全部流量”的数据库更新和变更日志没有放在同一个事务中。
- TLS 迁移遇到异常 JSON 或证书记录时会跳过该行，但版本仍可能继续推进。
- 老客户端的 `createdAt` 没有回填，界面会继续显示 `0` 或 `–`。

## 11. 后续同步检查清单

1. 先确认上游最新稳定 tag，不要直接以 `main` 为发布基线。
2. 分别确认后端和前端精确基线 SHA。
3. 继续把上游 submodule 前端与 Pro 内嵌前端分开处理。
4. 不要覆盖 Pro 的聚合订阅、限速、批量导入、批量编辑和主题功能。
5. 检查上游数据库迁移是否仍依赖版本号大小比较。
6. 检查 bcrypt、TLS、客户端统计字段和热更新逻辑的兼容性。
7. 运行后端测试、vet、前端类型检查和生产构建。
8. 严格按 `编译和git规范.md` 交叉编译、检查 ELF 和发布包内容。
9. 发布前确认使用 `pro` remote，避免推送到错误仓库。
10. 明确决定是否把发布分支合并回 `pro/main`。
