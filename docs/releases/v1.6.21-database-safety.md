# s-ui-pro v1.6.21 数据库安全发布记录

发布日期：2026-07-15

本文记录 `v1.6.21` 的数据库备份、还原、锁与运行态一致性修复。上游 `s-ui v1.5.3` 和 sing-box `1.13.14` 的同步内容已在 [`v1.6.20`](v1.6.20-upstream-merge.md) 完成，本版本不再变更 sing-box 版本。

## 1. 兼容性结论

- **没有修改数据库模型结构**：没有新增、删除或重命名业务表、字段、索引。
- 同版本数据库可以直接备份和还原。
- 导入旧版本数据库时，迁移只在上传的临时库中执行；迁移、初始化或完整性检查失败时不会触碰线上数据库。
- Linux Release 继续使用 CGO 构建，以支持 SQLite Online Backup API；non-CGO 构建会安全拒绝还原，而不是冒险覆盖数据库。

## 2. 备份与还原安全

- 备份改为 SQLite 原生 `VACUUM INTO` 一致快照，自动包含全部表、索引和触发器，修复旧实现遗漏 `services`、`tokens` 的问题。
- 备份在返回前执行完整性和核心表检查，临时目录权限为 `0700`，数据库文件和 CLI 备份文件为 `0600`。
- CLI 备份采用同目录临时文件加原子替换，避免覆盖失败留下半个数据库。
- Web 还原采用流式上传，默认限制 4 GiB，避免把整个文件一次性读入内存。
- 上传库会依次经过 SQLite 文件头、`integrity_check`、核心表和关键列、配置 JSON、迁移、初始化及 page size 检查。
- 通过预检后才使用 SQLite Online Backup API 提交；提交前保存旧线上库快照，复制或最终校验失败会在同一独占连接内回滚。
- 已提交但应用重载调度失败时会明确提示手动重启，不会把已经验证的新数据库再次回滚。

## 3. 数据库锁与一致性

- SQLite 连接池限制为单连接并保留 WAL 和 busy timeout，降低同进程内写锁竞争。
- WAL checkpoint 改为非阻塞 `PASSIVE`。
- cron 任务同步注册、禁止重入，停止时等待正在运行的任务退出，避免重载产生重复写库任务。
- 统计写入、用户流量重置和审计记录使用事务；失败时整体回滚，并把已取出的流量放回 tracker。
- inbound 与客户端配置热更新先完成校验和数据库提交，失败时恢复旧运行态，避免数据库与 sing-box core 状态不一致。
- 普通停止/重启会在关闭 core 前后刷新流量；数据库还原专用重启会丢弃旧运行态计数，避免污染刚恢复的新数据库。
- 迁移错误改为返回调用方，不再因 `log.Fatal` 或 panic 直接终止宿主进程。

## 4. 运维说明

- Web 备份默认排除 `stats` 和 `changes`；需要全量备份时必须取消这两个排除项。
- 还原前仍应保留一份独立的当前数据库备份，并确认磁盘有上传库、回滚快照及 WAL 所需空间。
- 数据库备份含 token、客户端凭据、配置密钥和密码哈希，应按敏感文件保存。
- 详细失败边界、环境变量和维护检查清单见 [`docs/database-safety.md`](../database-safety.md)。

## 5. 验证范围

发布前执行：

```bash
go test -count=1 ./...
go test -race -count=1 ./database ./cmd/migration ./core ./service ./cronjob
go vet ./...
CGO_ENABLED=0 GOOS=windows GOARCH=arm64 go build ./...
```

Linux amd64 Release 按 [`编译和git规范.md`](../../编译和git规范.md) 使用 glibc 交叉工具链、CGO 和 GNU tar 构建并检查压缩包内容。
