# S-UI 项目代码总结

> 本文档基于源码全量扫描产出，旨在让读者快速、完整地理解 **S-UI v1.4.1** 的所有功能、模块边界、关键流程与数据契约。
> 项目定位：基于 [sing-box](https://github.com/SagerNet/sing-box) 内核构建的高级 Web 管理面板，支持多协议、多用户、订阅服务与流量统计。

## 三、整体架构

```
┌────────────────────────────────────────────────────────────┐
│                          main.go                           │
│  无参 → runApp()；有参 → cmd.ParseCmd()（CLI 子命令）     │
└──────────────────────────────┬─────────────────────────────┘
                               │
                  ┌────────────▼─────────────┐
                  │         app.APP          │
                  │  Init() → Start() → Stop │
                  └──┬─────────────────────┬─┘
                     │                     │
        ┌────────────▼──────────┐  ┌──────▼────────────┐
        │   web.Server (2095)   │  │  sub.Server (2096)│
        │  Gin + Session + API  │  │  Gin + 订阅渲染   │
        └──────────┬────────────┘  └─────────┬─────────┘
                   │                          │
                   │ ┌────────────────────────┴────────────┐
                   │ │ service.* (ApiService 聚合所有服务) │
                   │ │ Setting/User/Client/Tls/Inbound/    │
                   │ │ Outbound/Endpoint/Services/Config/  │
                   │ │ Panel/Stats/Server/Warp             │
                   │ └─────────────┬──────────────────────-┘
                   │               │
                   │   ┌───────────▼──────────┐
                   │   │  database (GORM + WAL)│
                   │   └───────────┬──────────┘
                   │               │
   ┌───────────────▼─────────────┐ │
   │       core.Core             │◀┘  增量管理 sing-box 适配器
   │  ├ Box（sing-box 实例）     │
   │  ├ StatsTracker（读写计数） │
   │  └ ConnTracker              │
   └─────────────────────────────┘
                ▲
                │
   ┌────────────┴───────────┐
   │  cronjob.CronJob       │
   │  stats / deplete /     │
   │  checkCore / delStats /│
   │  WAL checkpoint        │
   └────────────────────────┘
```

### 关键观察
1. **sing-box 以库形式加载**：核心通过 `core.Core.Start([]byte)` 把 JSON 配置反序列化为 `option.Options`，然后实例化封装后的 `Box`（`core/box.go`）。增删 Inbound/Outbound/Endpoint/Service 都会调用 sing-box 各适配器 Manager 的 `Add/Remove`，**无需重启进程**即可热更新。
2. **HTTP 入口分离**：面板（Web）与订阅（Sub）使用独立 Gin 实例与端口；都支持 HTTPS（监听器使用 `network.AutoHttpsListener` 自动嗅探 HTTP→HTTPS）。
3. **守护型 cronjob**：当核心因配置错误未启动时，`CheckCoreJob` 每 5 秒尝试启动，保证服务自愈。
4. **配置变更广播**：每次保存通过 `ConfigService.Save` 写 `Changes` 表，并刷新内存中 `LastUpdate`，前端轮询 `api/load?lu=` 做增量拉取。

## 五、数据模型（SQLite + GORM）

`database/db.go` 调用 `AutoMigrate` 注册以下 11 张表：

| 表 | 主要字段 | 用途 |
| --- | --- | --- |
| `users` | `id, username, password, last_logins` | 后台管理员；默认 `admin/admin`，明文比对（仅本地内网） |
| `tokens` | `id, desc, token, expiry, user_id` | apiv2 调用凭据，支持永久 / 到期；token 通过 `common.Random(32)` 生成 |
| `settings` | `key, value` | 38+ 配置项（webPort、webPath、secret、subPort、config、订阅扩展模板等） |
| `tls` | `name, server(json), client(json)` | TLS 配置模板，由 Inbound/Service 通过 `tls_id` 引用 |
| `inbounds` | `type, tag, tls_id, addrs(json), out_json(json), options(json)` | 入站；`Options` 存 sing-box 原生字段；`OutJson` 存反推得到的“客户端用”出站模板；`Addrs` 多地址（CDN/Reality 等） |
| `outbounds` | `type, tag, options(json)` | 出站；默认带一条 `{type:direct, tag:direct}` |
| `endpoints` | `type, tag, options(json), ext(json)` | WireGuard / Tailscale / WARP 端点；`Ext` 用于 Warp 注册返回值 |
| `services` | `type, tag, tls_id, options(json)` | sing-box service（DERP / Resolved / SSM API / CCM / OCM） |
| `clients` | `id, enable, name, config(json), inbounds(json[uint]), links(json), volume, expiry, up, down, group, desc, delay_start, auto_reset, reset_days, next_reset, total_up, total_down` | 业务用户：单个 client 可绑定多个 Inbound；`Config` 内含各协议配置（uuid/password/method 等）；`Links` 为缓存的本地/外部/订阅链接 |
| `stats` | `date_time, resource, tag, direction, traffic` | 入站/出站/用户流量时间序列（写入间隔 10 秒） |
| `changes` | `date_time, actor, key, action, obj(json)` | 全量审计；前端通过此表做增量同步 |

> 关键 JSON 列：`Inbound`/`Outbound`/`Service`/`Endpoint`/`Client` 均实现了自定义 `MarshalJSON / UnmarshalJSON`，把固定列与动态 sing-box 选项合并/分离，保证 SQLite 列收敛但 sing-box 字段灵活。

## 六、核心模块详解

### 6.1 sing-box 内核集成（`core/`）

- **`core/register.go`** 注册所有协议、DNS、服务工厂到 sing-box Registry：
  - **入站协议**：tun、redirect、tproxy、direct、socks、http、mixed、shadowsocks、vmess、trojan、naive、shadowtls、vless、anytls、hysteria、tuic、hysteria2
  - **出站协议**：direct、block、selector、urltest、socks、http、shadowsocks、vmess、trojan、naive（条件）、tor、ssh、shadowtls、vless、anytls、hysteria、tuic、hysteria2
  - **Endpoint**：wireguard、tailscale（条件）
  - **DNS Transport**：tcp/udp/tls/https/hosts/local/fakeip/quic/h3/dhcp/tailscale
  - **Service**：resolved、ssmapi、derp（条件）、ccm、ocm
- **`core/box.go`** 封装 sing-box `Box`，并暴露 `AddInbound / RemoveInbound / AddOutbound / RemoveOutbound / AddEndpoint / RemoveEndpoint / AddService / RemoveService` 等热插拔方法。
- **`core/main.go`** `Core.Start([]byte)`：反序列化配置 → 创建 Box → 启动；`Stop` 关闭并清空指针；`IsRunning` 控制是否做热更新。
- **`core/tracker_stats.go`** `StatsTracker` 用 `atomic.Int64` 记录 inbound/outbound/user 三维度的读写字节；`RoutedConnection` 在 sing-box 路由阶段挂上 `Int64CounterConn`。`GetStats()` 把累积值 Swap 出来生成 `[]model.Stats`。
- **`core/tracker_conn.go`** `ConnTracker` 记录连接来源；`CloseConnByInbound(tag)` 在 Inbound 重启时强制断开旧连接，避免脏会话。
- **`core/outbound_check.go`** `CheckOutbound(ctx, tag, link)` 调用 sing-box `urltest.URLTest` 返回延迟（15 秒超时），用于面板「测速」按钮。
- **`core/register_naive*.go` / `register_tailscale*.go`** 提供条件编译的桩（无构建标签时退化为空实现）。

### 6.2 业务服务（`service/`）

`ApiService`（`api/apiService.go`）通过结构体嵌入聚合所有 Service，是 HTTP 处理函数的真正实现。

| Service | 主要职责 | 关键方法 |
| --- | --- | --- |
| `SettingService` | 38+ 配置项的读写；含默认值 map；首次访问自动写入；安全字段（secret/config/version）从 `GetAllSetting` 隐藏 | `GetAllSetting`、`Get/Set Port/Path/Cert*`、`GetFinalSubURI(host)`（拼接 protocol/host/port/path） |
| `UserService` | 管理员登录、修改密码、apiv2 token CRUD；`LoadTokens` 给 `apiv2Handler` 内存缓存 | `Login`、`ChangePass`、`AddToken/DeleteToken/GetUserTokens` |
| `ClientService` | 业务用户的 CRUD（含 `addbulk` / `editbulk` / `delbulk`）；自动重算 `Links`（本地链接 + 外部链接 + 订阅链接）；`DepleteClients` 实现到期/流量到顶禁用；`ResetClients` 实现延迟启动与周期重置 | `Save`、`UpdateClientsOnInboundAdd/Delete`、`UpdateLinksByInboundChange`、`DepleteClients`、`ResetClients`、`findInboundsChanges` |
| `TlsService` | TLS 模板 CRUD；编辑后联动刷新关联 Inbound / Service 并重启 | `Save`、`GetAll` |
| `InboundService` | Inbound CRUD + 热更新；自动给 sing-box 注入 `users` 字段（来自 `clients.config`）；`shadowtls v3+`、`shadowsocks 2022` 等特例分支；`RestartInbounds` 在用户/TLS/inbound 变更时调用 | `Save`、`addUsers`、`initUsers`、`fetchUsers`、`UpdateOutJsons`、`RestartInbounds` |
| `OutboundService` | Outbound CRUD + 热插拔 | `Save`、`GetAll` |
| `EndpointService` | Endpoint CRUD；`type=warp` 时自动注册 Cloudflare WARP（生成密钥、注册设备、抓取 v4/v6/对端公钥） | `Save`、`GetAll` |
| `ServicesService` | sing-box service CRUD + 热插拔 | `Save`、`GetAll`、`RestartServices` |
| `WarpService` | Cloudflare WARP API 封装（注册、修改 license、reserved 解析） | `RegisterWarp`、`SetWarpLicense` |
| `ConfigService` | **总指挥**：拼装完整 sing-box 配置（合并 Setting.config + 各资源），统一 `Save`（开启事务、写 `changes`、按对象类型分发到子 Service、必要时启核心）、`CheckChanges`、`CheckOutbound`、`StartCore/StopCore/RestartCore`；`config` 对象保存后异步调用 `restartCoreWithConfig` | `GetConfig(data)`、`Save(obj, act, data, initUsers, loginUser, hostname)` |
| `StatsService` | 周期从 `StatsTracker` 拉取数据写入 `stats` 表；下采样到 60 行（30 桶）返回给前端；`onlineResources` 内存缓存在线 tag | `SaveStats`、`GetStats`、`GetOnlines`、`DelOldStats` |
| `ServerService` | 系统监控聚合（cpu/mem/disk/diskIO/swap/net/host/sing-box 状态/DB 计数）；最近日志拉取；**密钥对生成**（ECH、TLS 证书、Reality、WireGuard） | `GetStatus(request)`、`GetLogs`、`GenKeypair`、`GetDatabaseInfo` |
| `PanelService` | 通过 SIGHUP/Kill 重启自身（延迟 3 秒） | `RestartPanel(delay)` |

> `corePtr` 是包级变量（`service/config.go` 中），由 `NewConfigService(core)` 注入，让其他 Service 能直接热操作 sing-box，避免循环依赖。

### 6.3 HTTP 入口（`api/` + `web/`）

- **`web/web.go`** 启动面板 HTTP，`embed.FS` 内嵌前端 dist；通过 `webPath` 拼接所有路由前缀；`NoRoute` 兜底重定向到 `/login` 或返回 SPA `index.html`；TLS 可选；监听器套 `AutoHttpsListener` 以同时接受 HTTP/HTTPS。
- **`api/apiHandler.go`** 面向浏览器，cookie session 鉴权（`s-ui` cookie）。路由：
  - **POST**：`login`、`changePass`、`save`、`restartApp`、`restartSb`、`linkConvert`、`subConvert`、`importdb`、`addToken`、`deleteToken`
  - **GET**：`logout`、`load`（全量/增量）、`inbounds`/`outbounds`/`endpoints`/`services`/`tls`/`clients`/`config`（部分）、`users`、`settings`、`stats`、`status`、`onlines`、`logs`、`changes`、`keypairs`、`getdb`、`tokens`、`singbox-config`、`checkOutbound`
- **`api/apiV2Handler.go`** 面向第三方/脚本，请求头 `Token: xxx` 鉴权；内存 `tokens` 缓存，过期自动剔除；`addToken/deleteToken` 后调用 `ReloadTokens` 刷新；接口集合比 v1 略少（无 login/logout/token 管理）。
- **`api/apiService.go`** 业务实现入口：
  - `LoadData` 根据 `lu`（lastUpdate timestamp）判断是否触发 `CheckChanges`，未变化时只返回 `onlines`，变化时一次性返回 config + clients + tls + inbounds + outbounds + endpoints + services + subURI + enableTraffic + onlines。
  - `Save` 转发到 `ConfigService.Save` 并部分回读。
  - `LinkConvert`（`POST /api/linkConvert`）：表单字段 `link` 传**单条节点 URI**（vmess/vless/trojan/hy/hy2/anytls/tuic/ss/naive+https/naive+quic/http2），走 `util.GetOutbound`，返回 1 个 sing-box outbound 对象，用于"出站"编辑器的"从链接导入"。
  - `SubConvert`（`POST /api/subConvert`）：表单字段 `link` 传**订阅 URL**（注意：是 URL，不是订阅正文！），走 `util.GetExternalSub`：
    1. 先 HTTP GET 该 URL（`InsecureSkipVerify=true`，跳过证书校验）；
    2. 调用 `StrOrBase64Encoded` 自动判断是否 base64，是则解码；
    3. **格式 A（sing-box JSON）**：内容以 `{` 起、`}` 止 → 必须包含 `outbounds` 数组，过滤掉 `urltest / direct / selector / block` 之后原样返回；
    4. **格式 B（多行 URI）**：否则按 `\n` 拆行，每行交给 `util.GetOutbound` 解析为 sing-box outbound 对象；
    5. 返回 outbound 对象数组，供前端批量勾选导入"出站"列表。
    > 常见误区：直接把订阅文本贴进 `link` 是不行的，**必须给一个能 GET 的订阅地址**；同时该 URL 必须返回 sing-box JSON 或 多行节点 URI（支持整体 base64），普通 Clash YAML 订阅不会被识别。
  - `ImportDb` 校验 SQLite 文件头 → 备份当前 DB → 替换 → 迁移 → 触发 SIGHUP 重启自身。
  - `GetDb` 把当前 DB 复制到临时文件，可选择排除 `changes` / `stats`，返回二进制下载流。
- **`api/session.go`** 用 `gin-contrib/sessions`（cookie store，使用 setting `secret`）；`SetLoginUser/ClearSession/IsLogin/GetLoginUser`；登录态以 cookie `s-ui` 存储，maxAge 来自 setting `sessionMaxAge`。

### 6.4 订阅服务（`sub/`）

- **入口**：`/sub/:subid`，`subid` 即 client name；HEAD 请求仅回写订阅响应头。
- **`SubService`**（普通格式）
  - 拉取 `enable=true AND name=subid` 的 client → 取 `Links`（local + external + sub） → 用 `LinkService` 渲染。
  - 可选 `subShowInfo`：在 vmess 的 `ps`、其他协议尾部追加剩余流量 / 剩余天数（emoji 风格）。
  - 可选 `subEncode`：base64 整体编码。
  - 响应头：`Subscription-Userinfo`（up/down/total/expire）、`Profile-Update-Interval`（小时）、`Profile-Title`（client name）。
- **`JsonService`**（`?format=json`，sing-box JSON 订阅）
  - 内置 tun + mixed 入站模板；
  - 对每个 client 关联的 Inbound 取 `OutJson`（FillOutJson 反推得到），按 `Addrs` 扩展为多地址；
  - 把 `mixed` 拆成 `socks` + `http` 两条；
  - shadowsocks 处理 `2022-blake3` 拼接用户密码；
  - 额外注入外部 link 转 outbound；
  - 默认追加 `selector(proxy) + urltest(auto) + direct(direct)`；
  - 路由模板（含 Clash Direct/Global 模式分支）可被 setting `subJsonExt` 覆盖（合并 log/dns/inbounds/experimental/rule_set/rules/default_domain_resolver）。
- **`ClashService`**（`?format=clash`，Clash.Meta 风格）
  - 接续 JsonService 的 outbound 集合，逐项转换为 Clash proxy 格式；
  - 支持 vmess/vless/trojan/tuic/socks/http/hysteria(1/2)/anytls/shadowsocks；
  - 处理 TLS（reality、ech、utls 指纹）、传输层（http/ws/httpupgrade/grpc）、Multiplex（含 brutal）；
  - 默认追加 `Proxy(select) + Auto(url-test)` 两个组；
  - YAML 模板可被 setting `subClashExt` 覆盖。
- **`LinkService`** 处理三种 link 类型：
  - `external` 直接透传 URI；
  - `sub` 调用 `util.GetExternalLink` 抓取并拆行；
  - `local` 仅在普通订阅下输出，可附加 ClientInfo。

### 6.5 链接生成与解析（`util/`）

- **`util/genLink.go` LinkGenerator** 根据 Inbound + Client.Config + Hostname 生成多种协议的标准 URI：socks、http、mixed、shadowsocks、naive、hysteria、hysteria2、tuic、vless、anytls、trojan、vmess。处理 Reality、ECH、ALPN、UTLS、Transport（ws/grpc/http/httpupgrade）、Multiplex、xtls-rprx-vision 自动剥离等细节。
- **`util/outJson.go` FillOutJson** 反向生成「客户端用 outbound JSON」缓存到 `inbound.out_json`，供订阅、Clash、JSON 订阅共用，避免每次重算。`tun/redirect/tproxy/direct` 跳过。
- **`util/linkToJson.go` GetOutbound** 解析多种外部 URI（vmess/vless/trojan/hy/hy2/anytls/tuic/ss/naive）为 sing-box outbound 对象，供「转换」按钮使用。
- **`util/subToJson.go` GetExternalSub** 抓取外部订阅 URL：自动 `StrOrBase64Encoded` 解码；若内容是 sing-box JSON（含 `outbounds` 数组），过滤 `urltest/direct/selector/block` 后原样返回；否则按 `\n` 拆行逐条调用 `GetOutbound` 转 outbound。**返回值都是 sing-box outbound 对象数组**，是 `/api/subConvert` 的底层实现。
- **`util/subInfo.go` GetHeaders** 拼出 subscription-userinfo 头。
- **`util/base64.go`** Padded base64 工具。
- **`util/common/`** 错误包装、随机字符串、`UnionUintArray` / `DiffUintArray`（用于检测 Client 绑定的 Inbound 变化）。

### 6.6 定时任务（`cronjob/`）

`CronJob.Start(loc, trafficAge)` 启动以下任务：

| 任务 | 调度 | 行为 |
| --- | --- | --- |
| `StatsJob` | `@every 10s` | 调用 `StatsService.SaveStats(enableTraffic)`：刷新在线 tag、累加 client 的 up/down，可选写入 `stats` 表 |
| `DepleteJob` | `@every 1m` | `DepleteClients` + 必要时 `RestartInbounds`：禁用到期 / 流量超额客户；同时执行 `ResetClients`（延迟启动启用、周期重置） |
| `DelStatsJob` | `@daily` | 仅在 `trafficAge>0` 时启用，删除 N 天前 stats |
| `CheckCoreJob` | `@every 5s` | 若核心未运行（配置错误/被关闭），自动尝试 `StartCore`；含 15 秒冷却 |
| `WALCheckpointJob` | `@every 10m` | `PRAGMA wal_checkpoint(FULL)` 强制 WAL 收敛 |

### 6.7 数据库备份/恢复（`database/backup.go`）

- `GetDb(exclude)`：把当前 DB 复制到临时 SQLite 文件，按需排除 `changes` / `stats` 表，返回字节切片供下载（`api/getdb`）。
- `ImportDB(file)`：
  1. 校验文件头 `SQLite format 3\x00`；
  2. 写入临时文件；
  3. 备份当前 DB 到 `.backup`；
  4. 用临时文件替换 DB；
  5. 调用 `migration.MigrateDb()` 升级；
  6. 失败回滚；
  7. 通过 `SendSighup` 让面板 3 秒后自重启加载新 DB。

### 6.8 数据库迁移（`cmd/migration/`）

- `migration.MigrateDb()` 比较 `settings.version` 与代码内 `config.GetVersion()`，依次执行 `to1_1 → to1_2 → to1_3` 升级脚本。
- 在 `entrypoint.sh` 与 `cmd/cmd.go` 的 `migrate` 子命令中均会触发。

### 6.9 网络层（`network/`）

`AutoHttpsListener / AutoHttpsConn` 在同一端口同时支持 HTTP 与 HTTPS：通过窥探首字节判断是否为 TLS ClientHello，自动给纯 HTTP 请求返回 301 升级到 HTTPS。

### 6.10 中间件（`middleware/`）

`DomainValidator(domain)` 校验 `Host` 头与 setting 中的域名一致，否则 403。Web 与 Sub 各自独立开关（setting `webDomain` / `subDomain`）。

### 6.11 日志（`logger/`）

包级 logger，记录在内存 ring buffer（用于 `GetLogs(count, level)` 给面板展示最近日志）；级别由 `SUI_LOG_LEVEL` 或 `SUI_DEBUG=true` 控制。

## 八、关键业务流程

### 8.1 启动流程
```
main.go → app.NewApp() → app.Init()
  ├ initLog()
  ├ database.InitDB(SUI_DB_FOLDER/s-ui.db)
  │   ├ AutoMigrate 11 张表
  │   └ 默认账号 admin/admin、默认 direct outbound
  ├ SettingService.GetAllSetting() （触发首次默认值落库）
  ├ core.NewCore() 注册 sing-box 工厂
  ├ cronjob.NewCronJob()
  ├ web.NewServer() / sub.NewServer()
  └ NewConfigService(core)
app.Start()
  ├ cron.Start(loc, trafficAge)
  ├ web.Start()（embed.FS + Gin）
  ├ sub.Start()
  └ ConfigService.StartCore() （读取 setting.config + 各资源 → sing-box）
```

### 8.2 单次保存流程（以 `clients` 为例）
1. 前端 `Data.save('clients', 'edit', payload)` → `POST api/save`。
2. `ApiService.Save` 解析表单，调用 `ConfigService.Save("clients", "edit", data, initUsers, loginUser, hostname)`。
3. 开启 GORM 事务：
   - `ClientService.Save` 写表；
   - 检测变化的 inbound id 列表；
   - 若需要，`InboundService.RestartInbounds(tx, ids)`：从 sing-box 移除旧 inbound → CloseConnByInbound → 重新注入带新 users 的 inbound。
4. 写一条 `Changes`（actor=登录账号，key=clients，action=edit，obj=原始 JSON）。
5. 提交事务，更新 `LastUpdate`；若 core 未运行则尝试 `StartCore`。
6. `LoadPartialData(["clients","inbounds"])` 回填给前端。

### 8.3 流量统计闭环
1. sing-box `RoutedConnection` → `StatsTracker.Counter.read/write` 原子累加。
2. 10 秒触发 `StatsJob`：`GetStats()` Swap 出累积值 → 维度（user）回写 `clients.up/down`，维度（inbound/outbound/user）填充 `onlineResources` → 可选写 `stats` 表。
3. 1 分钟触发 `DepleteJob`：根据 `clients.expiry / volume / enable / auto_reset / delay_start` 启用/禁用/重置；若启用变化，重启相关 inbound。
4. 每日触发 `DelStatsJob`：删除超过 `trafficAge` 天的 stats。
5. 前端通过 `/api/stats?resource=&tag=&limit=` 拉取下采样后的 30 桶数据。

### 8.4 订阅访问流程
1. 客户端访问 `https://host:2096/sub/<clientName>`（或自定义 path）。
2. `SubHandler.subs` 根据 `?format=` 路由到 `SubService` / `JsonService` / `ClashService`。
3. 普通订阅：合并 client.Links → 渲染 → 可选 base64 → 写订阅头返回。
4. JSON 订阅：模板 + outbound 集合 + 自定义扩展（`subJsonExt`）。
5. Clash 订阅：从 outbound 集合转换为 Clash proxies + groups，叠加 YAML 模板（`subClashExt`）。

### 8.5 WARP 一键注册
1. 前端新建 endpoint `type=warp`。
2. `EndpointService.Save` 检测到 type=warp 时调用 `WarpService.RegisterWarp`：
   - 本地生成 WireGuard 密钥；
   - 调用 `https://api.cloudflareclient.com/v0a2158/reg` 注册设备并获取 device_id/token/license/v4/v6/peer；
   - 计算 `reserved`（base64 → hex → []int）；
   - 把 access_token/device_id/license_key 存入 `endpoint.ext`，把 private_key/address/peers 存入 `endpoint.options`；
3. 后续若 license_key 修改，调用 `SetWarpLicense` PUT API 更新。

## 十、安全与运维要点

1. **明文密码**：用户密码以明文存储与比对（GORM where），仅适合本地内网 / Cloudflare Tunnel 后端。需要强对外暴露时建议套 Caddy / Nginx + WAF。
2. **Session 安全**：cookie store secret 默认随机 32 位，存于 settings 表，无前端可见接口；session 默认无 maxAge（浏览器关闭即失效），可在 setting 中改。
3. **TLS 证书**：面板与订阅可分别配置 `webCert/Key/Domain` 与 `subCert/Key/Domain`；`AutoHttpsListener` 自动给 HTTP 请求 301。
4. **API v2 Token**：写入数据库，HTTP 头 `Token: xxx`；面板查询时只显示 `****`；过期后内存缓存自动清理。
5. **域名校验**：开启 `webDomain` / `subDomain` 后 Host 头不匹配直接 403。
6. **审计日志**：所有保存操作落 `changes` 表（actor/key/action/obj）；前端 `/api/changes?a=&k=&c=` 查询。
7. **DB 备份**：面板「Backup/Restore」即 `GetDb / ImportDb`；导入时会自动建立 `.backup` 文件并在失败时回滚。
8. **WAL Checkpoint**：每 10 分钟强制收敛，并在长事务（如 `DepleteClients`）结束后立即 `PRAGMA wal_checkpoint(FULL)`，避免 WAL 文件膨胀。
9. **核心自愈**：`CheckCoreJob` 每 5 秒检测，并带 15 秒冷却防抖；配置错误时仅记录 `lastLog` 推给前端弹窗。
